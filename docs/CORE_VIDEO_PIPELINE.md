# 从「写好剧本」到「拿到视频」——用最直白的话说清楚

本文写给**不常做视频开发**的同学：尽量少术语，只讲你要做什么、算成功是什么样。

---

## 一句话：什么叫「整条链路跑通」？

你可以把它想成三件事连在一起：

1. **有一份讲清楚拍什么的说明**（系统里叫「剧本」，就是一个结构化的文字方案：分几段、每段说什么、画面大概怎样）。  
2. **电脑按这份说明去调度、画图、压成视频**（你在网页里点「运行」，或在命令行里执行一次任务）。  
3. **最后你能在文件夹里打开一个真正的 `.mp4` 视频文件**——能播放，文件不是空的。

做到第 3 步，就可以认为：**标准化生成流水线在你这台机器上已经能用了**。

---

## 和「智能助手 / Agent」是什么关系？

- **日常用法**：你在网页里用自然语言提需求，系统会帮你**整理成一份标准任务单 + 剧本 JSON**（这就是「专业脚本」那一环）。  
- **自动测试为了不花钱、不连外网**：仓库里放了一份示例剧本 [`static/sample_screenplay.json`](../static/sample_screenplay.json)，用来假装是助手已经生成好的结果——**流程和你真实使用是一样的**，只是剧本内容固定成示例。

你本地如果已经配置了 AI 接口，把助手生成的那份 JSON 换进去即可，**步骤不用改**。

## `VIDEO_PIPELINE_E2E=1` 是干什么的？（可选，给想自测的人）

这是 Windows 里一个**开关名字**（环境变量）。你可以先不管它，**不影响正常用网页做视频**。

只有当你想跑仓库自带的「真生成一段 mp4」的自动检查时，才需要打开它：

| 不开这个开关 | 平时跑测试会很快，**跳过**真正渲染视频（省时间、不占用你电脑做大片段编码）。 |
| 设为 `1` | 会真的在你电脑上编码出一段测试用的 mp4（一般要 **一两分钟**，且需要已安装 **FFmpeg**）。 |

**怎么开（复制到 PowerShell 里执行后再跑测试）：**

```powershell
$env:VIDEO_PIPELINE_E2E = "1"
.\.venv\Scripts\python.exe -m pytest tests/test_video_pipeline_e2e_render.py -v -m video_e2e
```

**前提**：命令行里能运行 `ffmpeg`（装过 FFmpeg 并加入系统路径即可）。

## 想自己点一点就验收：用网页（最推荐）

1. 启动控制台（例如双击 `run_dashboard.bat`），浏览器打开提示的地址。  
2. 左侧选好项目，点**成片类任务**（例如「生成最终视频」这类）。  
3. 按页面说明填参数；若使用剧本，按「脚本交付」导入或粘贴 JSON。  
4. 点运行，等进度走完，在**验收区**里会给出**成片路径**——到文件夹里双击打开 `.mp4` 即可。

**若只想确认「程序没坏、不接视频」**：可选任务里的 **「环境自检」**（`smoke_check`），几秒就出结果，**不会生成视频文件**。
