# 全局设置模块重构对比报告

**文档日期**: 2026-03-25  
**版本**: v2.3.0 → v2.3.1

---

## 📋 概述

原项目的**"全局设置"**模块在重构后进行了功能拆分和整合，部分功能保留，部分功能被新的架构替代。

---

## 🔄 功能对比总览

| 原功能模块 | 重构后状态 | 新位置/替代方案 |
|-----------|-----------|----------------|
| **AI 配置** (OpenAI API) | ✅ 保留 | 代码工坊 - AI 代码生成配置 |
| **视频参数配置** | ✅ 保留并增强 | 首页 - 4 步向导 (步骤 2: 填写数据) |
| **品牌主题配置** | ✅ 保留并增强 | 首页 - 4 步向导 (步骤 3: 选择品牌) |

| **画质选择** | ⚠️ 简化 | 整合到品牌主题中 |
| **全局默认值设置** | ❌ 移除 | 使用代码预设值 |
| **独立设置页面** | ❌ 移除 | 整合到各功能模块中 |

```html
<button class="btn btn-outline-secondary w-100 shadow-sm" data-bs-toggle="modal" data-bs-target="#settingsModal">

```

```
- APIConfig 类
- openai_compatible_base_url
- openai_compatible_api_key

```

### 2. 视频参数配置

- ✅ 与模板选择联动
- ✅ 实时数据验证

### 4. 渲染质量预设

**原位置**: `index_old.html` - 全局设置 - 画质选择

**重构后**: 
- **位置**: `shared/config/settings.py`
- **预设**:
  ```python
  QUALITY_PRESETS = {
      "preview": {"dpi": 72, "bitrate": 4000, "preset": "veryfast", "crf": 28},
      "draft": {"dpi": 108, "bitrate": 8000, "preset": "faster", "crf": 21},
  ```

**使用方式**: 
- 通过 API 参数传递
- 默认使用 `draft` 预设

### 5. 环境变量管理

**原位置**: `.env` 文件

**重构后**: 
- **位置**: `.env` + `shared/config/settings.py`
- **配置方式**: 
  ```bash
  # .env 文件
  VIDEO__WIDTH=1080
  VIDEO__HEIGHT=1920
  API__OPENAI_COMPATIBLE_BASE_URL=https://api.openai.com/v1
  API__OPENAI_COMPATIBLE_API_KEY=sk-xxx
  ```

- **代码访问**:
  ```python
  from shared.config.settings import settings
  
  # 访问配置
  width = settings.video.width
  api_key = settings.api.openai_compatible_api_key
  ```

## ⚠️ 简化的功能

### 1. 平台选择 (抖音/B 站等)

**原功能**: 全局设置中可选择输出平台 (抖音、B 站、YouTube 等)

**重构后**: 
- **状态**: ⚠️ 简化
- **新位置**: 整合到模板选择中
- **说明**: 不同模板自动适配不同平台规格
  - 竖屏模板 → 抖音/TikTok
  - 横屏模板 → B 站/YouTube

---

### 2. 画质选择

**原功能**: 全局设置中可选择画质 (流畅、高清、超清)

**重构后**: 
- **状态**: ⚠️ 简化
- **新位置**: 整合到品牌主题中
- **说明**: 品牌主题隐含画质预设
  - 默认/企业 → draft (1080p)
  - 金融时报 → final (1440p)

---

## ❌ 移除的功能

### 1. 全局默认值设置

**原功能**: 设置各项参数的全局默认值

**移除原因**: 
- 增加配置复杂度
- 用户很少使用
- 代码预设值已足够

**替代方案**: 
- 使用合理的代码预设值
- 通过模板继承默认配置

---

### 2. 独立设置页面

**原功能**: 独立的全局设置页面/模态框

**移除原因**: 
- 用户需要离开当前工作流
- 配置上下文丢失
- 增加学习成本

**替代方案**: 
- **情境化配置**: 在需要时自动提示配置
- **向导式流程**: 4 步向导整合所有配置
- **AI 辅助**: AI 编译页面自动检测配置状态

---

## 📊 架构对比

### 原架构 (v2.2 及之前)

```
全局设置模态框
├── AI 配置 (OpenAI API)
├── 视频参数
│   ├── 分辨率
│   ├── FPS
│   └── 画质
├── 品牌主题
├── 平台选择
└── 默认值设置
```

**问题**:
- ❌ 配置与工作流分离
- ❌ 需要单独打开设置页面
- ❌ 配置项过多，用户困惑

---

### 新架构 (v2.3+)

```
首页 (4 步向导)
├── 步骤 1: 选择模板
├── 步骤 2: 填写数据 (视频参数)
├── 步骤 3: 选择品牌 (品牌主题)
└── 步骤 4: 生成视频

代码工坊
└── AI 代码生成 (自动提示 AI 配置)

配置文件
├── .env (环境变量)
└── shared/config/settings.py (代码配置)
```

**优势**:
- ✅ 配置整合到工作流中
- ✅ 情境化提示，减少认知负担
- ✅ 向导式交互，降低学习成本
