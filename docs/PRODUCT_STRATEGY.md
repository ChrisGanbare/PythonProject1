# 产品战略：与「通用视频 AI」的差异与护城河

**安装、调度、API 与控制台**：见仓库根目录 [README.md](../README.md)。本文档只阐述战略与取舍，不替代 README 中的操作说明。

## 我们在比较什么

Claude、Gemini、以及各类「一句话出片」的模型，强项是：**意图理解、文案、分镜、画面风格、快速试错**。它们适合探索型创作、泛娱乐、轻知识类内容。

本仓库所代表的产品方向是：**数据可视化视频工程平台**——把 **可验证的数据、可复现的图表动画、平台规格化的成片** 串成流水线，服务 **财经、科普、增长对比、榜单赛跑** 等「数字必须站得住」的自媒体场景。

二者不是「谁更强」，而是 **解决的问题不同**。我们的优越性建立在 **不去做通用视频模型的替身**，而是做它们 **补不齐的一层**。

## 阻塞与对标：头部作品作参照系

设计与实现遇到取舍或瓶颈时，**不以「对话模型能生成什么」为上限**，而以 **数据可视化领域公认的高完成度作品** 为参照系——对标的是 **信息密度、标注清晰度、动效节奏、色彩与字体层级**，而非照搬题材或画面版权。

**可对照的方向（示例）**：动态图表与叙事节奏（如 Flourish、机构媒体数据短片常见的 **清晰轴标签 / 少而准的注解 / 渐进揭示**）；版式与阅读动线（如 **NYT / The Guardian / Financial Times** 等数据新闻的 **标题—副标—图源** 结构）；工程化可参考 **Observable / D3** 思路的 **数据绑定与可复现**，但本仓库以 Python 栈实现。

**落地要求**：产品/工程决策卡壳时，先问「头部作品在这种情况下会牺牲什么、强调什么」，再映射到我们的 **真值、可复现、平台规格** 三原则；避免用「模型随便能出」替代 **可审计的数据叙事**。

## 独有优势与价值（护城河）

### 1. 真值层（Truth Layer）：数字从代码与数据来，不从幻觉来

通用大模型视频容易在 **数字、单位、趋势** 上「看起来对」实则不可审计。本平台以 **Python 计算 + 明确数据源** 为单一事实来源；AI（剧本、旁白）只服务 **叙事与节奏**，不替代 **利息、还款、排名** 等可核验结果。  
**战略含义**：产品设计始终把 **数据/公式 → 渲染输入** 的链路做硬；剧本与画面绑定 **viz 规格与哈希**，可追溯。

### 2. 可复现性（Reproducibility）：同一输入 → 同一图表与成片参数

「再生成一版」在通用视频 AI 里往往是另一条随机轨迹。本平台追求 **版本化剧本、render manifest、质量档与平台预设**，便于 **纠错、合规复查、系列栏目统一风格**。  
**战略含义**：强化 manifest、定稿流、缓存键；不把「成片」只存一个 mp4，而存 **可重建该片的规格集合**。

### 3. 可控像素级可视化（Deterministic Viz）：Matplotlib 栈是工具，不是黑箱

通用模型对「图表长什么样」的控制是概率性的。我们用 **帧动画、主题 token、安全区**，适合 **品牌感稳定、版式可模板化** 的 MCN 与机构创作者。  
**战略含义**：坚持 `shared/visualization` 契约与后端可插拔；**不**把主路径改成「纯 prompt 出视频」。

### 4. 平台原生（Platform-Native）：B 站 / 抖音 / 小红书是一等公民

通用工具输出往往是「一个视频文件」。我们内建 **分辨率、帧率、时长区间、字幕安全区**，减少二次剪辑与违规裁切。  
**战略含义**：任何新功能默认问一句：**在哪个平台、什么规格**。

### 5. 系列化产能（Series Production）：模板 + 素材库 + 任务队列

自媒体高效来自 **同一结构、换数据、换钩子**。通用对话式 AI 不擅长替你维护 **栏目模板、素材复用、批量任务状态**。  
**战略含义**：剧本库、资产注册表、调度器与 API 任务继续作为 **增效核心**，而不是摆设。

## 我们刻意不主打什么（避免战略摇摆）

- **不**与「纯创意、无强数据约束的短片」生成拼画面想象力上限。  
- **不**把「替代剪辑师剪故事片」当作主卖点。  
- **不**暗示「不用核对数字」——相反，**核对数字要更容易**（摘要 API、与成片同源）。

若用户只需要「随便讲个故事、画面好看」，用通用视频 AI 往往更轻；若用户需要 **数据对、能复盘、能系列化**，才是本平台的主场。

---

## 锁定大方向后的设计原则（优越性策略）

| 原则 | 落地 |
|------|------|
| **真值优先** | 新增功能先接数据/schema，再接叙事；对外暴露「摘要/校验」接口 |
| **可复现优先** | 成片必带可追溯元数据；关键版本可「定稿」再批量出片 |
| **可控 viz 优先** | 默认 Matplotlib/明确后端；Plotly/Manim 为可选增强 |
| **平台规格优先** | UI/API 默认带 `platform`；禁止静默默认横屏等 |
| **系列化优先** | 模板、素材 id、任务状态可查询；减少一次性脚本 |
