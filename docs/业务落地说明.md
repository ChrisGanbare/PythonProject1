# 业务落地说明（给负责人 / 运维，非技术版）

本文说明：**这套系统在什么条件下能稳定给业务用**，以及**出问题先看哪里**。技术细节见 [DEPLOY.md](../DEPLOY.md)。

---

## 1. 系统能做什么（业务视角）

- 在浏览器里做 **数据图表类短视频**（首页向导）。
- 在控制台里管理 **项目、任务、成片作业**；作业结果会进数据库，**重启服务后仍可查**。
- 可选：接大模型做脚本/意图相关能力（需配置 API Key，不配则仅影响「需要 AI」的功能）。

---

## 2. 最少要准备什么才能「能用」

| 项 | 是否必需 | 说明 |
|----|----------|------|
| Python 3.9+ | 必需 | 与 README 一致 |
| `pip install -r requirements.txt` | 必需 | 装依赖 |
| **FFmpeg** 已安装且在系统路径里 | **成片必需** | 不做视频可暂不强调；做快速图表视频或渲染任务则必需 |
| 网络与端口 | 必需 | 内网控制台常用 **8090**；防火墙放行 |
| `.env` 配置文件 | 建议 | 复制仓库根目录 **`.env.example`** 为 **`.env`**，再按需改；不配也能启动，部分能力受限 |

---

## 3. 启动前先自检（推荐）

在项目根目录执行：

```bash
python main.py doctor
```

若输出「关键项通过」，再启动服务；若有 **✗ [必需]**，按提示处理（常见是未装 FFmpeg 或未装依赖）。

---

## 4. 怎么启动（复制给运维即可）

在服务器上进入项目根目录后：

```bash
# 日常：Web 控制台（同事用浏览器访问）
python main.py dashboard --host 0.0.0.0 --port 8090

# 或只当 API 用（与上面是同一套服务，默认端口不同）
python main.py api --host 0.0.0.0 --port 8000
```

若只输入 `python main.py`，程序会打印**下一步可选命令**，按提示选即可。

---

## 5. 怎么确认「已经正常」

在浏览器访问（把地址换成你的服务器 IP 和端口）：

1. 能打开首页（图表向导或控制台页）。
2. 技术同事可在同一地址后加 **`/docs`** 看接口文档。
3. 探活（可交给监控系统轮询）：  
   `http://<服务器>:<端口>/api/studio/v1/health`  
   应返回带 `"status":"ok"` 的 JSON。

**架构入口（机器可读，对接第三方时用）：**  
`http://<服务器>:<端口>/api/meta/architecture`

## 6. 常见问题（业务侧先自查）

| 现象 | 可能原因 |
|------|----------|
| 视频生成失败、日志里提 ffmpeg | 服务器未装 FFmpeg 或未加入 PATH |
| 端口起不来 | 端口被占用；换端口或结束旧进程 |
| 重启后作业没了 | 早期版本问题；当前版本作业在数据库，应检查是否删了 `runtime` 下库文件或改了数据库路径 |
| 某子项目接口 404 / 挂载失败 | 依赖未装全（如图像库）；技术同事看启动日志里 `Domain mount skipped` |
| AI 相关按钮不可用 | 未配置 `.env` 里 OpenAI 兼容接口与 Key |

---

## 7. 技术负责人维护原则（与业务对齐）

- **单一规则**：成片与作业统一走控制台同一套任务与数据库，避免「两套说法」。
- **优先保**：能启动、能探活、能完成一条最小成片路径；再谈大重构。
- **大改动**（例如把各子项目接口全部合并进一份 OpenAPI）单独立项，评估停机与对接成本后再做。
