控制台使用说明
侧栏：两大分区
  • 项目与任务：新建项目（磁盘脚手架）与刷新项目列表（重新请求 GET /api/registry）只出现在此分区。任务按类型分组（自检与预览 / 成片与导出 / 后台服务 / 其他），与主区「配置→执行→验收」是不同层级——前者是找入口，后者是单次运行的步骤。
  • 意图会话：新建会话对应 POST /api/studio/v1/sessions；在 POST /api/studio/v1/agent/compile 中携带 session_id 后，可在此查看历史与标准模板 JSON。顶栏仅保留「使用说明」，不再放置与 Registry 混用的刷新按钮。
主区：单次任务流程

选中任务后，卡片内步骤条表示当前这一次运行：配置参数与剧本 → 执行（后台 Job）→ 验收结果与成片路径。

长驻「启动 API」类任务请在终端执行，避免阻塞 Web 后台。
